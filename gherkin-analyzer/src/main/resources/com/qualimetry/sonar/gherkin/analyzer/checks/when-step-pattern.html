When steps should match the configured regular-expression pattern. This rule lets teams enforce naming conventions on action steps — for example, requiring them to start with a noun phrase such as “the” or “a”.

Noncompliant Code Example

With the pattern ^the .*:

Feature: Account management

  Scenario: Deactivate an account
    Given the user has an active account
    When account is deactivated                       <!-- Noncompliant -->
    Then the account status is inactive

Compliant Solution

Feature: Account management

  Scenario: Deactivate an account
    Given the user has an active account
    When the account is deactivated
    Then the account status is inactive

Parameters

NameDefaultDescription
pattern .* Regular expression that When step text must match.
