All Feature files in a project should use the same Gherkin language. Mixing languages (e.g., some features in English and others in French) reduces readability and consistency across the test suite. The first language encountered during analysis is treated as the expected language for all features.

Noncompliant Code Example

In login.feature (English):

Feature: User login
  Scenario: Login with valid credentials
    Given the user is on the login page
    When the user enters valid credentials
    Then the user is redirected to the dashboard

In commandes.feature (French):

# language: fr
Fonctionnalité: Gestion des commandes           <!-- Noncompliant: inconsistent language -->
  Scénario: Passer une commande
    Etant donné que le client est connecté
    Quand le client passe une commande
    Alors la commande est confirmée

Compliant Solution

Use the same language for all features:

Feature: Order management
  Scenario: Place an order
    Given the customer is logged in
    When the customer places an order
    Then the order is confirmed
