Examples tables in a Scenario Outline must contain at least one data row beyond the header row. An Examples table with only a header provides no test data, rendering the Scenario Outline useless. The header row defines the column names; data rows provide the actual values that are substituted into the outline steps.

Noncompliant Code Example

Feature: Currency conversion

  Scenario Outline: Convert between currencies
    Given the exchange rate from <source> to <target> is <rate>
    When the customer converts <amount> <source>
    Then the result should be <expected> <target>

    Examples:                                   <!-- Noncompliant -->
      | source | target | rate | amount | expected |

Compliant Solution

Add at least one data row to the Examples table:

Feature: Currency conversion

  Scenario Outline: Convert between currencies
    Given the exchange rate from <source> to <target> is <rate>
    When the customer converts <amount> <source>
    Then the result should be <expected> <target>

    Examples:
      | source | target | rate  | amount | expected |
      | USD    | EUR    | 0.85  | 100    | 85       |
      | GBP    | USD    | 1.27  | 50     | 63.50    |
