Features should not exceed a configurable number of scenarios. A feature file with too many scenarios becomes difficult to navigate, understand, and maintain. Large features often indicate that the file covers multiple business capabilities that should be split into separate, more focused feature files.

Noncompliant Code Example

With the default limit of 12 scenarios:

Feature: Complete e-commerce workflow            <!-- Noncompliant -->

  Scenario: Browse product catalog
    ...
  Scenario: Search for a product
    ...
  Scenario: View product details
    ...
  Scenario: Add product to cart
    ...
  Scenario: Update cart quantity
    ...
  Scenario: Remove product from cart
    ...
  Scenario: Apply discount code
    ...
  Scenario: Calculate shipping cost
    ...
  Scenario: Select payment method
    ...
  Scenario: Process credit card payment
    ...
  Scenario: Process bank transfer
    ...
  Scenario: Confirm order
    ...
  Scenario: Cancel order
    ...

Compliant Solution

Split the feature into smaller, domain-specific features:

Feature: Product browsing
  Scenario: Browse product catalog
    ...
  Scenario: Search for a product
    ...
  Scenario: View product details
    ...

Configuration

The maxScenarios parameter controls the maximum number of scenarios allowed in a single Feature. The default value is 12.

ParameterTypeDefault
maxScenariosint12
