Then steps should match the configured regular-expression pattern. This rule lets teams enforce naming conventions on outcome steps — for example, requiring them to start with a noun phrase such as “the” or “a”.

Noncompliant Code Example

With the pattern ^the .*:

Feature: Account management

  Scenario: Deactivate an account
    Given the user has an active account
    When the account is deactivated
    Then account status is inactive                   <!-- Noncompliant -->

Compliant Solution

Feature: Account management

  Scenario: Deactivate an account
    Given the user has an active account
    When the account is deactivated
    Then the account status is inactive

Parameters

NameDefaultDescription
pattern .* Regular expression that Then step text must match.
