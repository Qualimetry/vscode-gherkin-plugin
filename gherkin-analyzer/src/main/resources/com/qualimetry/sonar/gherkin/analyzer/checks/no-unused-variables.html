All columns in an Examples table should be referenced as <variable> placeholders in at least one step of the owning Scenario Outline. Unreferenced columns represent dead test data that add noise and can confuse readers about which values actually drive the test.

Noncompliant Code Example

Feature: User registration

  Scenario Outline: Register a new user
    Given the registration form is displayed
    When the user registers as a <role>
    Then the account is created

    Examples:                                <!-- Noncompliant: "unused" column not referenced -->
      | role    | unused   |
      | admin   | foo      |
      | editor  | bar      |

Compliant Solution

Remove unreferenced columns from the Examples table:

Feature: User registration

  Scenario Outline: Register a new user
    Given the registration form is displayed
    When the user registers as a <role>
    Then the account is created

    Examples:
      | role    |
      | admin   |
      | editor  |
